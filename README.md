# Ride-share metrics — a working showcase

Between **December 2025 and April 2026** I drove **890+ private-hire (Uber) trips across London** in an
electric vehicle. Alongside the driving I built a suite of software tools — using AI-assisted development
with **Claude Code** — to automate the paperwork and to answer two questions with data:

1. **Which trip offers are worth accepting?**
2. **When is it most productive to work?**

This repository is a self-contained demonstration of that tool suite and its real output. Nothing is
mocked: every chart, table and map is generated from the projects' own report files. Passenger names are
removed and addresses reduced to area level.

**▶ Browse the showcase:** open `docs/index.html` (or the GitHub Pages site if enabled — Settings →
Pages → deploy from `/docs`).

## The tool suite

| Tool | What it does | Stack |
|---|---|---|
| **Portal automation** (`uber`) | Drives a browser over the Uber portal; extracts every trip and weekly statement to JSON; audits trips against payments | Node.js, Jest |
| **Metrics pipeline** (`uber_screen`) | Reads Driver-app screenshots with AI vision, anchors them to GPS, reconciles against portal earnings, computes CNHR/MNHR cost-normalised hourly-rate metrics, renders dashboards and route maps | Python, Anthropic API |
| **Sentinel** (`uber_surge`) | Polls 10 TfL endpoints + National Rail + fixtures + events + weather into SQLite, testing whether ride demand can be anticipated from transport disruptions | Python, SQLite |
| **Ops heatmap** (`uber_heatmap`) | Interactive map dashboard of London disruptions, airport arrivals, events and weather | Python/Flask, Leaflet |
| **Charging parser** (`uber_charging`) | Parses EV charging invoices from four networks with Claude into per-session cost records | Python, Anthropic API |


## Headline numbers (all real)

- 931 portal activities · **£12,813** gross over 17 Uber weeks
- **318 GPS route segments** across 20 tracked days, mapped with per-segment distance, speed and earnings
- 762 screenshot-extracted trips matched to portal earnings; **6/17 weeks** hit the £15/h after-cost target
- 82 charging sessions · 3,540 kWh · **£1,428** — parsed from invoices, feeding the cost model
- One empirical finding that changed behaviour: **weekly hours, not trip quality, set the ceiling on profitability**
